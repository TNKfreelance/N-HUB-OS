# 27. Publishing Engine

## 公開先候補

note / GitHub / SNS など、成果物の性質に応じて選ぶ。特定チャネルへの自動投稿は実装しない(手動確認を挟む)。

## 公開前チェックリスト

- [ ] 引用・参照元が明記されているか(著作権配慮)
- [ ] 機密情報・個人情報が含まれていないか
- [ ] `brain/verified/` の裏取りが済んだ内容のみを根拠にしているか(未検証情報を断定的に書いていないか)
- [ ] `brain_approval_check` を実行し、リスクtierを確認したか

## 承認フロー

公開は原則 tier2 以上として扱い、`brain_approval_check` の結果に関わらず、最終的な「公開する/しない」の判断はオーナーの明示承認を得る(N-Link Hard invariant 3: 「The user decides」)。

## 公開ログ

保存先は実装時に確定させる(現時点で先取りして仕様化しない)。公開日・チャネル・対象成果物・承認者を記録する。

```markdown
| 日付 | チャネル | 成果物 | 承認 |
|---|---|---|---|
```

## 現状

公開実績なし。Publisher skill・公開ログの運用は初回の公開依頼が来た時点で開始する。
