# MCPの2026年最新動向まとめ(仕様のステートレス化とレジストリの実運用)

## リード文

Model Context Protocol(MCP)は2026年、2つの大きな動きを見せている。ひとつはプロトコル仕様そのものの刷新(2026-07-28リリース候補)、もうひとつは公式レジストリの実運用開始だ。両方とも「MCPが実験段階から実運用インフラへ移行しつつある」ことを示している。

## 本文

**仕様のステートレス化**。2026-07-28に最終公開予定のMCP仕様は、プロトコル発足以来最大の改定となる。これまで必要だった `initialize` ハンドシェイクやセッションIDによる接続管理を廃止し、どのサーバーインスタンスでも任意のリクエストを処理できる完全ステートレスな設計に変わる。これにより、MCPサーバーを標準的なロードバランサ配下でスケールさせやすくなる。あわせて、OAuth 2.0のResource Indicatorsの実装が必須化されるなど、認可まわりのセキュリティも強化される。

**レジストリの実運用**。仕様面の変化と並行して、MCPサーバーを発見するための公式レジストリ(`registry.modelcontextprotocol.io`)は、単なる構想ではなく実際に稼働しているサービスであることが確認できた。実際にAPIへアクセスすると、稼働中の実サーバーの情報(バージョン・エンドポイント・更新日時など)を含むJSONが返ってくる。現在はまだプレビュー段階(破壊的変更の可能性あり)だが、「MCPサーバーのアプリストア」としての土台はすでに機能している。

この2つの動きを合わせて見ると、MCPは「各社が個別に実装するプロトコル」から「共通のインフラとして運用される標準」へと重心を移しつつあると言える。
